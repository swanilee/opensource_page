Get Started

Visual Studio 를 활용하여 작성


총 23 단계로 구성되며, 사진아래에 세부 설명이 적혀 있습니다.

1.

[ new repository 만들기 ]

먼저 clone을 위한 repository를 만듭니다.

2.

[ 간단한 c 코드 작성 ]

clone을 하기 위한 c코드를 작성합니다.

3.

[ visual studio 상에서 clone을 합니다. ]

clone을 하기 위한 c코드를 작성합니다.

4.

[ clone 완료 ]

clone을 하기 위한 c코드를 작성합니다.

5.

[ 새 솔루션 만들기 ]
6.

[ 내용 변경 후 commit ]

7.

[ 동기화 ]

위 상단에 보이는 동기화 버튼을 눌러줍니다.


8.

[ remote 확인 ]
9.

[ fetch 후 merge 단계 시작 ]

기존의 원격 저장소 사진




위 상단에 보이는 동기화 버튼을 눌러줍니다.


[ gitignore 변경 ]

gitignore 변경후 second commit -> third commit 으로 변경되었습니다.


[ gitignore의 변경된 내용 ]

10.

[ branch 만들기 ]

branch명은 hotfix로 하였습니다.




hotfix branch 로 설정한 모습입니다.


11.

[ branch checkout ]

master branch -> hotfix branch 로 checkout한 모습입니다.


[ 동기화 후 브랜치 생성 확인 ]

12.

[ 기존 코드내용 변경 ]

기존의 내용에 한줄을 더 추가해주었습니다.


13.

[ hotfix branch에서 코드수정 commit ]



* 동기화에 주의하여야합니다!!
( 충돌 발생을 생각한다면 동기화를 누르지 않도록 하세요 )


14.

[ 병합 ]

현재 분기와 기본분기의 충돌이 일어날 병합을 시도합니다. 하지만.. !




생각과는 다르게 충돌이 일어나지 않아
당황하여 다시 gitignore을 확인하였습니다.


15.

[ 재시도 ]

hotfix에서 다시 수정을 하여 commit을 하였습니다.
중간에 확인을 해가며 이후 2번의 commit ,
병합 시도 끝에 충돌이 발생하였습니다.


16.

[ 충돌 발생 ]



충돌이 발생되는 것이 이렇게 기쁠 수 없었습니다.
메시지를 확인하고, "충돌비교" 를 시작합니다.


17.

[ 충돌 내용 확인 ]



어디서 충돌이 일어났는지 보여줍니다.
이제 병합을 눌러 충돌을 해결합니다.


18.

[ 충돌 해결 ]

충돌 부분을 해결하였습니다.
이제 커밋 과 동기화를 해주려고 합니다.


19.

[ 충돌 해결 후 커밋 과 동기화 부분 ]

병합 완료 메세지를 적은 후,
"스테이징 된 항목 커밋" 이라고 적힌 버튼 옆 화살표 부분을 눌러
선택지를 변경해 커밋과 동기화를 동시에 진행하게끔 합니다.




처리가 성공적으로 완료 되었습니다!
20.

[ 원격 repository 에서의 확인 ]

"병합 완료." 라고 적었던 커밋 메세지도 잘 보이고,
위에 노란 푸쉬 알림을 보아 잘 된것 같습니다.


21.

[ 마지막, Pull 기능 ]

캡처를 하기전 한번 시도해본것을 포함하여
pull이 성공적으로 되었습니다 :)
