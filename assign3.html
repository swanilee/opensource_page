Pix2pix Opensource Training


총 14 단계로 구성되며, 사진아래에 세부 설명이 적혀 있습니다.
코드 편집기로는 visual code를 사용하였습니다.

1.

[ 설치 환경 ]


# OS : Linux
# Python : 3.7.1
# GPU : NVIDIA GEFORCE 940MX
# CUDA : CUDA CuDNN 7.3.1

Anaconda VirtualEnvironment에서 패키지를 설치하였습니다.
다음은 제가 설치한 가상환경 패키지 목록 캡처 이미지입니다.


저의 경우 readme 파일을 읽고, 설치를 하였지만,
cudnn을 불러오지못해 다시 설치를 해주었습니다.
또한 이후에 계속해서 일련의 패키지 오류가 발생하여 검색을 해본결과
정확히는 나와있지 않았지만, pip update를 진행하고 numpy를 직접 다시 설정을 해주니 오류가 해결되었습니다. :)

# 가상환경 리스트 보기 : conda env list

# 가상환경 활성화 : conda activate MyEnvironmentName

# 가상환경 비활성화 : conda deactivate

# 가상환경 패키지 내보내기 명령어 : conda list --explicit > spec-file.txt ( ~.txt 에서 ~부분에는 원하시는 파일명을 입력하시면 됩니다.)

# 가상환경을 새로 만들면서 패키지 설치 : conda create --name MyEnvironmentName --file spec-file.txt

# 기존의 가상환경에 패키지 설치 : conda install --name MyEnvironmentName --file spec-file.txt
2.

[ Pix2pix train/test Access ]

python train.py 이후 "--example" 이러한 옵션들은
개인적으로 커스텀 할 수 있습니다.

학습 결과나 손실률을 쉽게 보기위한 시각적효과 기능을 제공하는데, 그 기능을 사용하려면 학습지시 코드 실행전 python -m visdom.server 코드를 입력합니다. 이후, http://localhost:8097 에서 볼 수 있습니다.

뿐만아니라 ./results/데이터셋명/test_latest/index.html 에 들어가면 scripts 를 확인할 수 있습니다.


학습에 들어가기 앞서, 나의 경우 저 학습코드를 실행하였었을때 cuda out of memory 오류를 받았습니다.

자세한 이유는 알 수 없지만 학습코드에 gpu 옵션을 추가해주지않으면 cpu로 동작이 되기에 발생하는 오류 같았습니다.

우선 gpu를 사용하여 학습을 진행하기 위해서는 저 코드에 하나의 옵션을 추가해주어야합니다.


# GPU 동작 여부 명령어 옵션 : --gpu_ids -1


위의 명령어를 입력하였을때 정상적으로 작동하였습니다.

이부분의 내용은 tips에서 참고하였습니다.
3.

[ 동작, 결과 확인 ]

위와 같이 설정해준 옵션을 한번 쫙 보여주고 학습이 진행되며 visdom.server에서는 학습되는것이 시각적으로 보여집니다. ( 옵션을 따로 지정해주지 않았다면 모두 default 값으로 진행됩니다 )
